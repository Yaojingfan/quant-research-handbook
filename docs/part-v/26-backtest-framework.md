# 26 回测的基本框架

> 所属模块：Part V 回测体系

> **回测的目标不是画一条向上的净值曲线，而是回答五个问题：有没有效、钱从哪来、风险多大、能不能交易、能不能复现。** 答不全，等于没做。

## 本节导读

本章搭建 A 股中低频多因子策略的 **回测骨架**：框架选型、流程、调仓规则与现金处理。强调与 Part II 数据口径、Part IV 组合规则 **严格一致**。

## 学习目标

1. 明确回测必须回答的五类问题
2. 选择向量化 vs 事件驱动回测
3. 实现完整的信号—组合—成交—收益闭环
4. 正确处理调仓频率、信号日与成交日错位

## 核心概念

回测 = **在历史数据上模拟** 策略若被真实执行时的持仓与损益，且模拟规则 **不得优于** 实盘可获得的条件。

---

## 26.1 回测要回答什么

| 问题 | 对应分析 | 失败信号 |
|------|----------|----------|
| 策略是否有效？ | 净收益、IR、显著性 | OOS 为负 |
| 收益来自哪里？ | 归因：行业/风格/选股 | 全是 Size Beta |
| 风险多大？ | 波动、回撤、TE | MDD 不可接受 |
| 是否可交易？ | 成本后净收益、成交率 | 成本吞噬毛 Alpha |
| 是否稳健？ | 子样本、参数敏感性 | 仅 IS 有效 |

**反炒作**：回测 Sharpe > 2 在 A 股多因子中并不罕见；**扣除成本、OOS、容量后的 Sharpe** 才是讨论对象。

---

## 26.2 向量化回测与事件驱动回测

### 向量化回测（Vectorized）

- **做法**：矩阵运算一次性算全样本持仓与收益
- **优点**：极快；适合因子检验、分组回测、月频指增
- **缺点**：复杂成交规则（涨跌停、部分成交）表达困难
- **适用**：中低频、规则清晰的 A 股多因子 **研究阶段**

### 事件驱动回测（Event-driven）

- **做法**：按时间事件推进：数据更新 → 信号 → 下单 → 成交 → 记账
- **优点**：贴近生产；易插入订单簿级逻辑
- **缺点**：慢；工程量大
- **适用**：模拟盘对接、复杂执行、多策略调度

### 选型建议

| 阶段 | 推荐 |
|------|------|
| 单因子 / 合成因子 | 向量化 |
| 指增 prototype | 向量化 + 成本层 |
| 准生产 / 模拟盘 | 事件驱动 |
| 高频 | 事件驱动 + Tick（本手册不展开） |

**诚实结论**：80% 的 A 股多因子研究用 **向量化 + 严格规则** 足够；事件驱动价值在 **执行一致性**，不在 Sharpe 提升。

## 26.3 回测流程

```mermaid
flowchart LR
    A[输入数据 PIT] --> B[计算因子/信号]
    B --> C[股票池过滤]
    C --> D[生成目标权重]
    D --> E[模拟交易与成本]
    E --> F[更新持仓/现金]
    F --> G[计算收益与风险]
    G --> H[归因与报告]
```

### 各步要点

1. **输入数据**：Point-in-Time；与 Part II 对齐
2. **信号**：含 lag；与因子 spec 一致
3. **股票池**：第 21 章规则
4. **目标权重**：第 23—25 章方法 + 第 24 章约束
5. **模拟交易**：第 27 章成本与成交约束
6. **持仓更新**：权重 drift、分红、公司行动
7. **绩效**：第 28 章指标
8. **稳健性**：第 29—30 章偏差审计与样本外验证

### 状态变量

每个交易日维护：

- $w_{i,t}$：权重
- $C_t$：现金
- $NAV_t$：净值
- 可选：空头、对冲 leg

## 26.4 调仓与持有期

### 频率

| 频率 | 典型场景 | 成本 |
|------|----------|------|
| 日频 | 短周期 Alpha | 高 |
| 周频 | 动量、资金流 | 中 |
| 月频 | 价值、质量、指增 | 低 |

A 股 **印花税（卖出）+ 最低佣金** 使日频全换仓极难盈利。

### 信号日与成交日

- **信号日 $t$**：收盘后数据齐备，计算因子
- **成交日 $t+1$**：开盘价 / VWAP / 收盘价 —— **三选一，全项目统一**
- **错误**：$t$ 日收盘信号 + $t$ 日收盘成交 → Look-ahead

### 推荐默认（中低频）

```
信号：T 日收盘后
成交：T+1 日 VWAP 或开盘价
成本：双边佣金 + 卖出印花税 + 滑点
```

### 持仓重叠

- 重叠持有：调仓仅交易 **Δw**，非全量买卖
- 换手定义：$\sum_i |w_{i,t} - w_{i,t-1}^{+}|$，$w^{+}$ 为 drift 后权重

---

## 26.5 现金、分红与公司行动

### 分红

- **除息日**：价格跳变已由复权处理；持仓股数不变则 **总权益不变**
- **现金分红**：增加现金账户；再投资规则需明确（当日 vs 下次调仓）

### 拆股 / 配股

- 用 **复权因子** 调整历史价格与股数，保持 continuity
- 配股若参与与否影响回测路径—— **默认不参与** 并文档化

### 现金管理

- 指增：**满仓** 假设，现金 ≈ 0（除分红暂存）
- 中性：**多头 + 空头 / 期货** 保证金；现金占比较低
- 无法买入（涨停）：产生 **现金拖累** —— 应建模，非强制满仓

## 数学定义

组合日收益（简化，无成本）：

$$
r_{p,t} = \sum_i w_{i,t-1} \cdot r_{i,t}
$$

含成本：

$$
r_{p,t}^{\mathrm{net}} = r_{p,t} - \mathrm{TC}_t
$$

Drift 后权重：

$$
w_{i,t}^{+} = \frac{w_{i,t-1}(1 + r_{i,t})}{\sum_j w_{j,t-1}(1 + r_{j,t})}
$$

## Python 示例

```python
import pandas as pd
import numpy as np

def backtest_weights(
    weights: pd.DataFrame,      # index=date, columns=symbol；信号日收盘目标权重
    returns: pd.DataFrame,      # 同 shape；次日（成交日）收益
    tc: pd.Series | None = None,
) -> pd.Series:
    """简化向量化回测。

    约定：`weights` 在日期 t 表示信号日 t 收盘形成的目标权重，
    于 t+1 成交并赚取 `returns` 在 t+1 的收益 → 用 shift(1) 对齐。
    勿把 weights 解释为「已是成交日开盘权重」后再 shift（会双重滞后）。
    """
    port_ret = (weights.shift(1) * returns).sum(axis=1)
    if tc is not None:
        port_ret = port_ret - tc
    return port_ret
```

## 常见错误

1. **信号成交同日** → 未来函数
2. **向量化忽略 drift** → 换手与成本低估
3. **分红未进现金** → 指增长期偏差
4. **回测与因子研究用不同 lag** → IC 与 PnL 不一致
5. **无 baseline** → 无法判断增量

## 要点回顾

- 回测五问：有效、来源、风险、可交易、稳健
- 中低频研究：向量化 + 严格 lag 与成本通常足够
- 信号日 ≠ 成交日；drift、分红、现金不可忽略
- 框架服务于 **证伪**，不是服务于 **推销曲线**

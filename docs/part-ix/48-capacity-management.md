# 48 策略容量与规模管理

> 所属模块：Part IX 从研究到实盘

> **Alpha 不是无限的；当资金规模超过市场能承受的换手时，回测曲线只是幻觉的延长线。**

## 本节导读

某小盘多因子策略回测年化 20%，AUM 从 2 亿扩到 20 亿后超额大幅衰减 — 不是 Alpha 突然消失，而是**冲击成本与流动性约束**非线性生效。本章定义容量、估算方法与规模控制手段。

## 学习目标

1. 理解策略容量的定义与影响因素
2. 会用成交额比例、冲击模型做粗算
3. 掌握封盘、规模控制与 AUM 分配原则

## 48.1 容量是什么

**定义**：在给定成本预算（如冲击 + 滑点 ≤ Y bp）和约束下，策略可承载的**最大 AUM**，且超额收益不明显衰减。

$$
Capacity \approx \min_i \left( \frac{V_i \cdot \phi}{w_i} \right)
$$

| 符号 | 含义 |
| --- | --- |
| $V_i$ | 股票 i 日均成交额 |
| $\phi$ | 可接受参与度（如 5%–10%） |
| $w_i$ | 组合权重 |

上式是「**假设当日可按目标权重全量换仓**」的上界示意；中低频策略应按 **turnover × AUM** 估日常冲击，容量常高于「每日全换」假设。容量由**最窄的那只股票**决定（木桶效应）。

---

## 48.2 流动性约束

**参与度规则**（示例）：

- 单日买入 ≤ 20 日平均成交额的 5%
- 组合调仓涉及的总成交额 ≤ 市场可吸收量

**小盘策略**：CSI1000 外股票占比高 → 容量显著低于大盘指增。

| 股票池 | 相对容量（示意） |
| --- | --- |
| 沪深 300 | 高 |
| 中证 500 | 中 |
| 中证 1000 | 中低 |
| 全市场 micro | 低 |

---

## 48.3 冲击成本

简化平方根模型：

$$
Impact \propto \sigma \cdot \sqrt{\frac{Q}{V}}
$$

$Q$ 为订单量，$V$ 为 ADV。规模翻倍，冲击并非线性翻倍 — **扩 scale 需重新估成本**。

回测中应使用 Part V 27.3 冲击函数，并按 AUM scale 压力测试。

---

## 48.4 规模扩大后的收益衰减

**衰减来源**：

1. 冲击与滑点上升
2. 无法成交的 alpha 边缘票增多
3. 因子拥挤（同类策略同步交易）
4. 被迫提高市值暴露，偏离最优因子组合

**监控**：滚动 `marginal IR vs AUM` 曲线；拐点即容量上限参考。

---

## 48.5 封盘与规模控制

| 手段 | 说明 |
| --- | --- |
| 硬封盘 | 产品不再接受新申购 |
| 软封盘 | 仅接受赎回或限量申购 |
| 内部 cap | 单策略 AUM 上限 |
| 复制账户 | 同信号多账户分散下单（有限帮助） |
| 降频 | 周频替代日频，降换手 |

**PM 决策**：接近容量 80% 时启动评估 — 扩 research universe 向上迁移 vs 封盘保 Alpha。

## Python 示意

```python
def estimate_capacity(weights, adv, participation=0.05):
    """weights: Series indexed by symbol, sum to 1."""
    cap_per_name = adv * participation / weights
    return cap_per_name.replace([np.inf, -np.inf], np.nan).min()
```

## 常见错误

- 回测无成交量约束，任意规模可交易
- 只用平均滑点，不做 scale stress
- 容量按最好股票算，忽略权重最大的 illiquid 名
- 封盘过晚，已有资金大幅回撤
- 忽视同类策略 crowding 对容量的侵蚀

## 要点回顾

- 容量是策略的物理极限，不是 marketing 数字
- 规模管理 = 流动性 + 冲击 + 拥挤 + 产品结构的综合题

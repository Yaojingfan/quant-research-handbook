# 46 策略生产流程

> 所属模块：Part IX 从研究到实盘

> **生产日的节奏是：数据先于信号，信号先于组合，组合先于订单 — 任何跳步都是事故预演。**

## 本节导读

某交易日 19:30 因子任务失败，但优化器仍用昨日因子跑完 — 次日组合暴露漂移 2 个标准差。本章描述 A 股多因子产品标准生产日节奏（nightly/daily **运行**）及岗位接口。

> **注意**：生产可按日更新数据与监控，但本手册策略主线为**中低频**——通常仅在调仓日做组合优化与下单；日频 Pipeline ≠ 日频调仓。

## 学习目标

1. 掌握从数据更新到盘后归因的完整日频流程
2. 明确研究、数据、PM、交易、风控各节点交付物
3. 理解失败时的阻塞与降级规则

---

## 日频 Pipeline

```mermaid
flowchart LR
    A[数据更新] --> B[因子计算]
    B --> C[目标组合]
    C --> D[风险检查]
    D --> E[订单生成]
    E --> F[交易执行]
    F --> G[盘后归因]
```

---

## 46.1 每日数据更新

| 数据 | 来源 | 校验 |
| --- | --- | --- |
| 日行情 | 交易所/供应商 | 行数、缺失、极值 |
| 复权因子 | 同左 | 与昨日连续性 |
| 财务增量 | 公告 | PIT 时间戳 |
| 指数成分 | 指数公司 | 生效日 |
| 停复牌/ST | 公告 | 状态机 |

**失败策略**：行情未齐 → **阻塞**因子与下游；非关键另类数据缺失 → 告警 + 因子降级。

---

## 46.2 因子计算

- 从 Factor Registry 读取定义与版本
- 增量计算 T 日截面
- 输出至 Factor Store + QA 报告（覆盖率、分布）

```text
factor_store/roe_ttm/v1.2/trade_date=20240315/
```

---

## 46.3 目标组合生成

1. 读取最新因子分数
2. 综合/优化得到 raw weights
3. 应用约束（行业、市值、换手、权重上限）
4. 输出 `target_portfolio.csv`

**接口字段**：`symbol, target_weight, trade_date, strategy_id, version`

---

## 46.4 风险检查

见 [35.1 事前风险检查](../part-vi/35-risk-monitoring.md) — 不通过则：

- 重新优化（放松/收紧约束）
- 或回滚至 T-1 目标
- 人工 sign-off 记录

---

## 46.5 订单生成

$$
order_qty = target_shares - current_shares
$$

- 整手取整（A 股 100 股）
- 过滤不可交易（停牌、涨停买、跌停卖）
- 输出至 OMS 格式

---

## 46.6 交易执行

- 算法单：VWAP / TWAP / POV
- 交易员监控异常成交
- 收盘后回传 `fill` 至数据库

---

## 46.7 盘后归因

- 实际持仓 vs 目标
- 当日 P&L 与超额初算
- 执行成本分解
- 更新滚动 IC、TE、暴露监控

报告推送 PM / 风控 / 研究 — 字段口径与 [40.6](../part-vii/40-research-platform.md) 对齐。

---

## 时间表（示例）

| 时间 | 任务 | 负责 |
| --- | --- | --- |
| 15:30–17:00 | 行情入库 | 数据工程 |
| 17:00–19:00 | 另类/财务 | 数据工程 |
| 19:00–21:00 | 因子 | 研究/开发 |
| 21:00–22:00 | 组合+风控 | PM/风控 |
| 22:00 | 订单文件 | 开发 |
| 次日 9:30–15:00 | 执行 | 交易 |
| 15:30+ | 归因 | 全员 |

---

## 常见错误

- 因子未更新仍调仓
- 目标组合无版本号，无法审计
- 订单未考虑整手与涨跌停
- 归因用目标权重而非实际成交
- 无 on-call，夜间失败次日才发现

## 要点回顾

- 生产流程的核心是**依赖、校验、版本、阻塞**

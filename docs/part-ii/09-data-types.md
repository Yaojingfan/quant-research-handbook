# 09 量化数据的基本类型

> 所属模块：Part II 数据是量化研究的起点

**因子是数据的函数——你连字段含义都没对齐，就别谈 Alpha。**

## 本节导读

新人入职第一天常被丢进 Wind、聚宽或内部数据库，面对上千张表不知从何下手。本章按 **A 股中低频多因子研究** 的实际需求，梳理行情、基本面、资金面、指数与另类数据的定位、典型字段与使用边界。

情景：你要构造一个「北向资金 + 低估值」复合因子。你需要：日频 OHLCV 算市值、Wind 北向持股算资金流、财报 ROE 与 PE 算价值——三类数据更新频率、对齐方式、缺失模式完全不同。先认清类型，再谈 merge。

## 学习目标

1. 区分行情、基本面、资金面、指数与另类数据的研究用途
2. 理解 A 股特有字段（复权因子、涨跌停、ST 状态）的含义
3. 知道高频/逐笔数据在本手册中的边界定位
4. 评估另类数据的合规与可得性风险

## 09.1 行情数据

行情数据（Market Data / Price Data）是量化研究的 **高频接触面**——几乎每一个因子最终都要与价格、成交量或市值发生关系。

### 核心字段

| 字段 | 英文 | 含义 | A 股注意点 |
| --- | --- | --- | --- |
| 开盘价 | Open | 当日首笔成交价 | 集合竞价后确定 |
| 最高价 | High | 当日最高成交价 | |
| 最低价 | Low | 当日最低成交价 | |
| 收盘价 | Close | 当日最后一笔成交价 | 回测信号常用收盘价 |
| 成交量 | Volume | 成交股数 | 单位：股（非手）需确认 |
| 成交额 | Amount / Turnover Value | 成交金额 | 流动性因子常用 |
| 前收盘价 | Pre-close / Prev Close | 昨收（未复权） | 涨跌幅计算基准 |
| 换手率 | Turnover Rate | 成交量 / 流通股本 | 注意流通股本口径 |
| 复权因子 | Adjustment Factor | 除权除息调整系数 | 详见第 12 章 |
| 交易状态 | Trading Status | 正常 / 停牌 / 涨跌停 | 可交易性判断 |

### 日频 vs 分钟频

本手册聚焦 **日频及以上** 的中低频研究。日频 OHLCV 足以支撑：动量、反转、波动率、流动性、市值等主流因子。分钟频数据用于日内策略或精细滑点建模，不在主线范围。

### 收益率计算

简单收益率（Simple Return）：

$$r_t = \frac{P_t}{P_{t-1}} - 1$$

对数收益率（Log Return）：

$$r_t^{log} = \ln\left(\frac{P_t}{P_{t-1}}\right) = \ln(1 + r_t)$$

A 股研究中两者皆可；横截面排序（Rank）时差异极小，时间序列累乘时需统一口径。

```python
import pandas as pd
import numpy as np

# 假设 df 含 symbol, trade_date, close_adj（后复权收盘价）
df = df.sort_values(["symbol", "trade_date"])
df["ret"] = df.groupby("symbol")["close_adj"].pct_change()
df["log_ret"] = df.groupby("symbol")["close_adj"].transform(lambda s: np.log(s).diff())
```

### 行情数据的「隐藏字段」

供应商常附带但新人易忽略：

- **limit_up / limit_down**：涨跌停价格，回测成交约束必需
- **is_suspended**：停牌标志，不可当缺失价格处理
- **list_date / delist_date**：上市退市日期，股票池构建必需
- **st_flag**：ST / *ST 状态，风控与股票池过滤

## 09.2 高频与逐笔数据

### 数据层级

| 层级 | 英文 | 粒度 | 典型用途 |
| --- | --- | --- | --- |
| Tick | Tick Data | 逐笔成交 | 微观结构、滑点 |
| 委托 | Order Book / Quote | 买卖盘口 | 流动性、价差 |
| Level 2 | L2 Market Data | 十档行情 | 高频策略 |
| 逐笔成交 | Trade-by-Trade | 每笔成交明细 | 订单流分析 |

### 本手册的定位

**概念了解，不作为核心。** A 股中低频多因子（指数增强、市场中性）的主线是日频 OHLCV + 基本面 + 资金面。Tick/L2 数据：

- 成本高、存储大、清洗复杂
- 对日频调仓策略的 Alpha 贡献有限
- 对执行层（Execution）更有价值——Part IX 会涉及

若团队做高频或 T0，需单独建数据栈；勿与日频 Research Layer 混用。

## 09.3 基本面数据

基本面数据（Fundamental Data）来自上市公司 **三大财务报表** 及衍生指标，是价值、质量、成长因子的原材料。

### 三大报表

| 报表 | 英文 | 核心内容 | 更新频率 |
| --- | --- | --- | --- |
| 利润表 | Income Statement | 营收、净利润、毛利率 | 季报 / 年报 |
| 资产负债表 | Balance Sheet | 资产、负债、净资产 | 季报 / 年报 |
| 现金流量表 | Cash Flow Statement | 经营/投资/筹资现金流 | 季报 / 年报 |

### 常用衍生指标

| 类别 | 指标 | 英文 | 典型因子 |
| --- | --- | --- | --- |
| 盈利能力 | ROE, ROA, 毛利率 | Profitability | 质量因子 |
| 成长能力 | 营收增速、利润增速 | Growth | 成长因子 |
| 偿债能力 | 资产负债率、流动比率 | Leverage / Solvency | 风险过滤 |
| 估值 | PE, PB, PS, EV/EBITDA | Valuation | 价值因子 |

### A 股基本面数据的特殊性

1. **报告期 vs 公告日**：2024Q3 报告可能在 10 月底才公告——按报告期 merge 是未来函数（详见第 13 章）
2. **财报重述（Restatement）**：历史财报可能被修正，需 PIT 数据库
3. **非标准审计意见**：ST 预警、退市风险的重要信号
4. **业绩预告 / 快报**：早于正式财报，信息增量但口径不一

```mermaid
flowchart LR
    A[报告期 Report Period] --> B[公告日 Announcement Date]
    B --> C[实际可得日 Available Date]
    C --> D[研究员使用 Research Use]
    D --> E[信号日 Signal Date]
    E --> F[成交日 Trade Date]
```

**铁律**：基本面因子必须按 **公告日或可得日** 对齐，绝不能按报告期 end_date 直接 merge 到交易日。

## 09.4 资金面与持仓数据

资金面数据反映 **市场参与者行为**，在 A 股因制度与投资者结构特殊而尤为重要。

### 主要类型

| 数据 | 英文 | 来源 / 特点 | 研究用途 |
| --- | --- | --- | --- |
| 北向资金 | Northbound Flow | 沪深港通持股变动 | 外资偏好、情绪 |
| 融资融券 | Margin Trading | 交易所公布 | 杠杆情绪、拥挤度 |
| 主力资金 | Main Force Flow | 供应商估算 | 争议较大，需验证 |
| 股东持仓 | Shareholder Holdings | 十大股东、十大流通股东 | 筹码集中度 |
| 基金持仓 | Fund Holdings | 季报披露 | 机构偏好、抱团 |

### 北向资金示例

北向资金（陆股通）是 A 股最具代表性的外资 proxy：

- **持股数量 / 占流通股本比**：日频更新（T+1 披露）
- **净流入金额**：可构造「北向动量」「北向反转」类因子
- **注意**：2024 年起披露频率与口径有调整，历史回测需确认数据版本

### 基金持仓的 PIT 问题

基金季报披露 **十大重仓股** 有严重滞后：

- Q1 报告 4 月底才全部披露
- 按披露日使用是 PIT；按报告期 end_date 使用是未来函数
- 适合低频、长持有期因子，不适合日频调仓

### 数据质量警示

「主力资金」「大单净流入」等字段 **缺乏统一定义**，不同供应商差异大。使用前务必：

1. 阅读供应商字段说明
2. 做与换手、成交额的相关性 sanity check
3. 在 Research Spec 中写明数据来源与版本

## 09.5 指数与成分股数据

指数数据（Index Data）定义了 **股票池（Universe）** 与 **基准（Benchmark）**——指数增强、中性策略的核心约束来源。

### 关键概念

| 概念 | 英文 | 说明 |
| --- | --- | --- |
| 成分股 | Constituents | 指数包含的股票列表 |
| 权重 | Weight | 自由流通市值加权等 |
| 样本空间 | Universe | 指数选样范围（如中证全指） |
| 调整 | Rebalance / Reconstitution | 定期换入换出成分股 |

### A 股主要指数

| 指数 | 代码 | 特点 | 典型用途 |
| --- | --- | --- | --- |
| 沪深 300 | 000300.SH | 大盘蓝筹 | 基准、增强 |
| 中证 500 | 000905.SH | 中盘 | 基准、增强 |
| 中证 1000 | 000852.SH | 小盘 | 小盘策略基准 |
| 中证全指 | 000985.CSI | 全市场 | 全 A 研究股票池 |

### 历史成分股的重要性

**致命错误**：用「当前沪深 300 成分股」回测 2015 年的策略。

- 当年不在指数内的「今日明星股」会被错误纳入
- 当年在指数内、现已退市的股票会被遗漏
- 结果：幸存者偏差 + 前视偏差，回测超额被系统性高估

**正确做法**：维护 **历史成分股快照（Historical Constituents）**，每个 rebalance 日记录当时成分与权重。Wind、CSMAR 等提供；自建需按指数公司公告逐期归档。

```python
# 错误：用当前成分过滤历史
current_hs300 = get_current_constituents("000300.SH")
history = prices[prices["symbol"].isin(current_hs300)]  # 错！

# 正确：按日 join 历史成分
history = prices.merge(
    index_constituents[["trade_date", "symbol", "weight"]],
    on=["trade_date", "symbol"],
    how="inner",
)
```

## 09.6 另类数据

另类数据（Alternative Data）指 **非传统行情与财报** 的信息源，近年来在量化研究中热度上升，但 A 股生态与美股差异大。

### 常见类型

| 类型 | 示例 | 潜在 Alpha | A 股现状 |
| --- | --- | --- | --- |
| 新闻 / 舆情 | 财经新闻 NLP 情绪 | 事件驱动、情绪因子 | 数据商较多，质量参差 |
| 搜索指数 | 百度指数、微信指数 | 关注度 proxy | 免费源有限 |
| 供应链 | 上下游关系图谱 | 产业链传导 | 需专业数据库 |
| 招聘 | 招聘量、岗位结构 | 扩张/收缩信号 | 合规与覆盖度问题 |
| 卫星 / 地理 | 停车场、工厂开工 | 实物 activity proxy | 成本高，A 股覆盖有限 |

### 使用边界与合规

1. **合规（Compliance）**：内幕信息、非公开重大信息严禁用于交易；另类数据的「公开性」需法务确认
2. **覆盖度（Coverage）**：A 股 5000+ 标的，另类数据往往只覆盖头部，因子覆盖率不足
3. **时效性（Timeliness）**：数据从产生到入库的延迟是否可接受
4. **可复现性（Reproducibility）**：供应商停服或改口径后，历史研究能否复现
5. **过拟合风险**：另类数据维度高、样本少，更易 p-hacking

### 本手册建议

- 新人 **优先掌握** 09.1～09.5 的传统数据
- 另类数据作为 **进阶拓展**，需有明确经济逻辑 + 严格 OOS
- 任何另类因子上线前，过合规评审

## 常见错误

- 把供应商字段名当含义——「净利润」可能是归母、扣非或 TTM，不读 Data Dictionary 必踩坑
- 基本面按报告期 merge 到交易日——未来函数头号来源
- 用当前指数成分回测历史——幸存者偏差 + Universe Leakage
- 忽视交易状态字段，把停牌日记为价格缺失并填 0
- 追逐另类数据而忽视传统数据口径——地基未稳就盖楼

## 要点回顾

- 行情（OHLCV + 状态字段）是因子计算的**基础输入**；注意复权与交易状态
- 基本面是价值/质量/成长因子之源；**必须 PIT 对齐**，按报告期 merge 是禁忌
- 资金面（北向、两融、持仓）在 A 股有独特价值；注意披露滞后与定义差异
- 指数成分股必须取 **历史快照**，不可用当前名单回测过去
- 另类数据合规优先，新人先把传统五类数据吃透

# 08 回测与实盘的距离

> 所属模块：Part I 认识量化研究

**回测是理想化的历史重演；实盘是带摩擦、带延迟、带情绪的真实市场——两者差距不是 bug，是 feature。**

## 本节导读

几乎所有量化事故，都始于「回测太好了」。不是回测无用——而是回测默认省略了太多实盘约束。本章系统梳理 **回测乐观偏差的来源**、**实盘摩擦**、**策略失效机制**，以及 **从回测到实盘的验证链条**。这是 Part I 的收官章，也是 Part V（回测框架）和 Part IX（生产）的情绪铺垫。

## 学习目标

1. 识别回测乐观偏差的来源
2. 建立从回测到实盘的验证链条意识

---

## 08.1 为什么回测看起来更好

回测引擎是 **有意简化** 的历史模拟器——简化本身没错，错在 **把简化结果当真相**。机构内部常要求：任何对外材料中的回测数字，须附 **假设清单**（成交、成本、池、滞后）。

回测（Backtest）是在历史数据上模拟策略表现。它 **系统性偏向乐观**，原因包括：

| 偏差来源 | 机制 | 典型后果 |
| --- | --- | --- |
| 成交价格理想化 | 假设按收盘价/开盘价全额成交 | 高估收益 1%～3%/年 |
| 成本低估 | 忽略滑点、冲击、融券费 | 高换手策略失真严重 |
| 信号立即可用 | 未考虑数据发布滞后 | 前视偏差，IC 虚高 |
| 数据完整 | 历史库已修复，无缺失 | 实盘遇停牌、延迟 |
| 容量约束缺失 | 假设无限流动性 | 大资金无法复制 |

### 情景：小王的完美回测

小王做了一个日频反转因子，回测 2018～2023 年化 25%，Sharpe 2.1。实盘三个月年化 8%。差异归因：

1. 回测用 **收盘价** 成交，实盘只能 **次日开盘** 成交——隔夜 gap 吃掉部分收益。
2. 未扣成本：若按「单次换手综合成本约 1.5%」（佣金+印花税+滑点+冲击），年化换手 400% → 成本拖累约 **6%/年**；若只计双边佣金类 0.3%、不计冲击，则约 **1.2%/年**——口径必须写清。
3. 回测未过滤 **涨跌停买不到** 的标的——实盘约 15% 信号无法执行。
4. 2023 年后 **微盘拥挤**，同类策略同步调仓，冲击上升。

四条叠加，25% → 8% 并不意外—— **不是因子完全失效，是回测口径太美**。

## 08.2 实盘中的真实摩擦

A 股量化须**显式**建模的摩擦：

| 摩擦类型 | 说明 | A 股特点 |
| --- | --- | --- |
| 手续费 Commission | 券商佣金 + 印花税 | 印花税 0.05%（卖出） |
| 滑点 Slippage | 成交价偏离预期 | 小票、开盘滑点大 |
| 冲击成本 Market Impact | 大单推动价格 | 容量核心约束 |
| 涨跌停 Limit Up/Down | 无法成交 | 10%/20%/ST 5% |
| 停牌 Suspension | 无法买卖 | 重组、异常波动 |
| 延迟 Latency | 信号到下单的时间差 | T+1 加剧 |
| 流动性 Liquidity | 成交量不足以承接 | 微盘、冷门行业 |
| 订单拒绝 Rejection | 风控、额度、系统错误 | 生产环境常见 |

### 成本的数量级直觉

| 策略类型 | 年化换手 | 粗略成本拖累 |
| --- | --- | --- |
| 低频价值 | 50%～100% | 0.5%～1.5% |
| 中频多因子 | 200%～400% | 2%～6% |
| 高频 | >1000% | 成本主导，须专门建模 |

**规则**：换手越高，回测与实盘差距越大—— **先建模成本，再汇报超额**。

```python
# 伪代码：简单成本模型
turnover = abs(w_new - w_old).sum() / 2
cost = turnover * (commission + slippage + impact(turnover, adv))
net_return = gross_return - cost
```

## 08.3 策略为什么会失效

策略失效（Strategy Decay）不等于「做错了」——市场、竞争、数据都在变。

| 失效原因 | 机制 | 应对 |
| --- | --- | --- |
| 市场结构改变 Regime Change | 政策、投资者结构变化 | 子样本监控、Regime 识别 |
| 因子拥挤 Crowding | 同类策略过多，Alpha 被挤掉 | 监控因子相关性、容量 |
| 数据口径变化 | 供应商改算法、复权调整 | 版本 diff、重跑历史 |
| 成本上升 | 竞争加剧、流动性下降 | 降换手、改执行 |
| 过拟合 Overfitting | IS 调参过多 | OOS 纪律、简化模型 |
| 模型漂移 Model Drift | 因子预测力衰减 | 滚动 IC、再训练 |
| 执行偏差 Implementation Shortfall | 回测与实盘成交差异 | 模拟盘、执行分析 |

### 因子拥挤的信号

- 同类因子产品 AUM 快速上升
- 因子值极端分布收窄（大家都在买同样的票）
- 调仓日冲击成本异常升高
- 滚动 IC 趋势性下降但逻辑未变

**诚实面对**：没有因子永远有效；问题是 **多快识别、多快响应**。

## 08.4 从回测到实盘的验证链条

不要把「回测过了」直接等同「可以上实盘」。标准链条：

```mermaid
flowchart LR
    A[样本内研究 IS] --> B[样本外验证 OOS]
    B --> C[Walk-forward]
    C --> D[模拟盘 Paper]
    D --> E[小资金实盘 Pilot]
    E --> F[正式运行 Production]
```

| 阶段 | 验证什么 | 通过标准（示例） |
| --- | --- | --- |
| IS | 想法与初筛 | IC 显著、逻辑清晰 |
| OOS | 未参与调参的时间段 | IC 衰减 < 50%，仍显著 |
| Walk-forward | 滚动样本外 | 多段 OOS 稳定 |
| 模拟盘 | 实时数据、模拟成交 | 与回测偏差可解释 |
| 小资金实盘 | 真实成本、真实冲击 | 超额在预期区间内 |
| 正式运行 | 规模化、监控 | 风控限额内、归因清晰 |

### 每阶段的核心问题

1. **IS → OOS**：是过拟合还是真规律？
2. **OOS → Walk-forward**：是单段运气还是跨期稳健？
3. **Walk-forward → 模拟盘**：是数据延迟还是执行问题？
4. **模拟盘 → 小资金**：是成本假设还是容量问题？
5. **小资金 → 正式**：是规模效应还是策略本身？

**跳过阶段 = 跳过问题**。模拟盘看起来「慢」，但比实盘交学费便宜得多。

### 回测质量自检清单

- [ ] 成交价格：信号日 T，成交日 T+1 开盘（或团队统一规则）
- [ ] 成本：含佣金、印花税、滑点、冲击（至少敏感性分析）
- [ ] 股票池：历史成分股，非当前成分
- [ ] 数据滞后：财务、持仓、指数调整按公告日
- [ ] 涨跌停 / 停牌：无法成交则跳过或顺延
- [ ] 容量：单票权重 × AUM < 日均成交额某比例
- [ ] OOS：独立时间段，未用于调参

### 模拟盘 vs 回测：多出来的检验项

| 维度 | 回测 | 模拟盘 |
| --- | --- | --- |
| 数据 | 历史批量 | 实时或准实时 |
| 成交 | 规则假设 | 模拟撮合 + 可接入真实盘口 |
| 系统 | 离线脚本 | 生产 pipeline 同路径 |
| 发现的问题 | 逻辑、口径 | 延迟、漏更新、权限、调度 |

**典型模拟盘 bug**：因子表更新完成时间是 09:25，但组合优化任务 09:20 已跑——全天用昨日因子。这类问题回测永远发现不了。

### 何时该放弃一条线？

不是每条研究都要上线。建议 **Kill Criteria**：

- OOS IC 低于团队底线（如 0.02）且持续 6 个月
- 中性化后分组 spread 不显著
- 扣贴近实盘的成本后 IR < 0.5
- 与现有策略相关 > 0.8 且边际 Alpha 不足
- 容量 < 产品最低规模

及时放弃与及时上线同样重要—— **机会成本** 是研究员最稀缺的资源。

### 偏差可接受区间（经验谈）

| 策略类型 | 回测 vs 实盘年化偏差 | 若超出则 |
| --- | --- | --- |
| 低频价值 | ±1%～2% | 查成本与成交规则 |
| 中频多因子 | ±3%～5% | 查冲击、涨跌停、延迟 |
| 高换手 | >5% 常见 | 必须模拟盘验证 |

超出区间触发 **归因流程**，不是自动关停。保存每一次偏差归因记录——这是团队级知识资产。

## 常见错误

- 回测曲线是唯一 KPI，不做模拟盘和小资金验证。
- 成本设为 0 或固定 0.1%，不随换手和流动性变化。
- 发现实盘偏差后改回测参数「对齐实盘」——应改的是策略或放弃，不是美化历史。
- 把策略失效全部归因于「市场不好」，不做拥挤与执行分析。
- 模拟盘用历史 replay 而非实时 pipeline，掩盖延迟 bug。

## 要点回顾

- 回测系统性乐观：理想成交、低成本、无延迟、无容量约束。
- A 股实盘摩擦：手续费、滑点、冲击、涨跌停、停牌、T+1、流动性。
- 策略失效来源：Regime、拥挤、数据变更、过拟合、执行偏差——需监控与响应。
- 验证链条：IS → OOS → Walk-forward → 模拟盘 → 小资金 → 正式；不可跳步。
- 回测不是谎言，但须 **保守假设、多层验证、持续对齐实盘**。
